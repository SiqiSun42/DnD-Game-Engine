### 第一步：列出所有常见剧情物品

| 序号 | 物品类型 | 典型例子 | 核心特征 |
|------|---------|---------|---------|
| 1 | **信件/密函** | 国王的密信、间谍的情报 | 有文字内容，需要阅读，交给指定 NPC |
| 2 | **日记/笔记** | 冒险者的日记、法师的研究笔记 | 有文字内容，提供线索或背景故事 |
| 3 | **地图/藏宝图** | 通往宝藏的羊皮纸 | 有内容（路线描述或绘制），指引地点 |
| 4 | **信物/徽章** | 王室徽章、佣兵团令牌 | 无阅读内容，交给特定 NPC 证明身份 |
| 5 | **钥匙** | 生锈的钥匙、水晶钥匙 | 无阅读内容，对特定锁/宝箱使用 |
| 6 | **容器/藏物** | 上锁的盒子、封印的卷轴筒 | 内部包含另一件物品，需要条件开启 |
| 7 | **组合碎片** | 圣杯碎片、符文石碎片 | 多件需收集齐全后组合成新物品 |
| 8 | **特殊工具** | 破城槌、龙裔号角、允许通过某地的通行证 | 在特定场合使用推动剧情 |

---

### 第二步：找出所有字段——覆盖 90% 剧情物品

从以上 8 类物品中抽象出以下字段，其中部分为可选（视物品类型而定）。

```json
{
  "tag": "剧情物品",
  "name": "物品名称",
  "quantity": 1,

  // 通用字段——所有剧情物品都有
  "description": "物品的外观和手感描述。",
  "task_hint": "（关联任务）给玩家看的提示，帮助回忆剧情。",
  "trigger_condition": "（触发条件）给AI看的判定依据，描述何时触发。",
  "triggered": false,
  "can_transfer": false,
  "weight": 0.1,

  // 可选字段——部分物品需要
  "content": null,         // 信件、日记、地图等有可读内容的物品使用
  "contains": null,        // 容器类物品（如宝箱、卷轴筒）内部包含的物品列表
  "key_for": null,         // 钥匙类物品，描述它打开什么（如"哥布林大王的宝箱"）
  "composes_into": null,   // 组合碎片类，组合后变成什么物品（如"完整的圣杯"）
  "required_parts": null,  // 组合碎片类，组合需要哪些碎片（用于碎片本身）
  "usage_location": null,  // 特殊工具类，在哪个地点使用（如"龙裔号角在龙脊山谷使用"）
  "hidden_secret": null,   // 隐藏秘密类，玩家不知道的信息（如“地图方向实际上是错误的，会通往陷阱”），可以附带鉴定条件

  "other": null
}
```

---

### 第三步：八个类型对应填哪些字段

| 类型 | 必填字段 | 可选字段 |
|------|---------|---------|
| 信件/密函 | `content` | （无） |
| 日记/笔记 | `content` | （无） |
| 地图/藏宝图 | `content` | `usage_location`（建议对应地图上的目的地） |
| 信物/徽章 | （无额外） | （无） |
| 钥匙 | （无额外） | `key_for`（建议填写对应锁或宝箱） |
| 容器/藏物 | `contains` | `trigger_condition`（开启条件） |
| 组合碎片 | `composes_into`, `required_parts` | （无） |
| 特殊工具 | `usage_location` | （无） |

### 结论：这个结构能覆盖哪些类型？

| 类型 | 能否覆盖 | 说明 |
|------|---------|------|
| 信件/密函 | ✅ | `content` |
| 日记/笔记 | ✅ | `content` |
| 地图/藏宝图 | ✅ | `content` + `usage_location` |
| 信物/徽章 | ✅ | 不需要额外字段 |
| 钥匙 | ✅ | `key_for` |
| 容器/藏物 | ✅ | `contains` + `trigger_condition` |
| 组合碎片 | ✅ | `composes_into` + `required_parts` |
| 特殊工具 | ✅ | `usage_location` |
