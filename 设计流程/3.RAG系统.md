【前端】
用户对话窗口 / 战斗系统 / DM创作界面
        ↓
【决策层】（可能硬编码，也可能自判断）
"这个操作需要RAG吗？"
        ↓ 是
【调用RAG】
query_knowledge(
    query_text="具体问题",
    knowledge_base="选择的库",
    top_k=5
)
        ↓
【RAG引擎】
├─ 向量化问题
├─ 在指定知识库中搜索
├─ 返回相关segments
        ↓
【LLM处理】
把segments + 问题 + prompt 送给DeepSeek
        ↓
【输出】
最终回答给用户

【后台进程】(平行运行，不阻塞用户)
每当有新消息时：
├─ 记录到saves
├─ 分块
├─ 向量化
└─ 添加到game_memory
