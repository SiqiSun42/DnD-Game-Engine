## 一、消耗品的数据字段

标 `*` 的为必填，其余可选。

```json
{
  "标签": "消耗品",
  "名称": "治疗药水",
  "数量": 2,
  "简介": "一瓶装在玻璃瓶中的红色液体，散发着淡淡的草药香气。",
  "价格": {
    "基础价": 50,          // 单位：金币（gp），供AI参考
    "波动范围": 0.1    // 可接受的价格波动区间，超出需特殊理由
  },
  "效果": [
    {
      "类型": "恢复",
      "作用对象": "生命值",  // 生命值 | 法术位 | 属性值 | 攻击检定 | 豁免检定 | 技能检定 | 临时生命值 | 状态（如中毒），必填，具体影响什么
      "骰子": "2d4",                                     // 可选，骰子表达式
      "固定值": 2,                                         // 可选，固定增减值，渲染为2d4+2
      "修饰": "优势",                                     //  可选， 优势 | 劣势
      "加减值": "1",                                       // 可选，一个整数，可为负（也就是减值）
      "法术": "火球术",                                      // 可选，当类型为“施法”时填写法术名称
      "解除": "中毒",                          // 中毒 | 疾病 | 诅咒， 可选，当类型为“解除”时填写
      "特殊": "获得10点临时生命值",                           // 可选，上述字段无法覆盖的用纯文本描述
      "时间": { 
        "持续时间": 0     // 单位：分钟；0 表示一次性
          },
      "专注": false,
      "豁免": null,
      "描述": "药水入口即化，一股温暖流过全身。"
    }
  ],
  "使用条件": {
    "动作类型": "动作",
    "目标": "自己",         // 自己 | 触及角色 | 距离15尺内可见生物等, 必填，描述目标范围
    "额外条件": "需要能够打开背包",
  },
  "稀有度": "常见",      // "常见", "非普通", "珍稀", "极珍稀", "传说"
  "重量": 0.5,           // 单位：磅（lb），便于负重计算
  "其他": null          //任何需要记录的其他重要信息
}
```

### 字段说明


| 字段     | 说明           | 为什么需要                           |
| ------ | ------------ | ------------------------------- |
| `名称`   | 物品的标准名       | AI 和玩家沟通时使用，保持统一                |
| `数量`   | 可堆叠          | 面包、药水通常可以堆叠，节省背包空间              |
| `简介`   | 一段短描述        | 供 DM/AI 在描述时参考，建议控制在 2 句以内      |
| `价格`   | 结构化价格        | AI 可以参考基础价进行交易，而非凭空定价           |
| `效果`   | **效果列表**（数组） | 这是核心，下面会说结构                     |
| `使用条件` | 何时能用         | 防止 AI 在战斗中让角色“花一个动作啃面包回血”的不合理情况 |
| `稀有度`  | 常见/非普通/稀有等   | 影响获取难度和价格上限                     |
| `重量`   | 负重计算用        | 让你的负重系统有据可依                     |

## 二、「效果」字段的结构化写法

```json
"效果": [
  {
"类型": "恢复 | 减益 | 伤害 | 施法 | 解除 | 变形 | 感知",  // 必填
"数值": "2d4+2",                                        // 可选，骰子或固定数值
"持续时间": "1小时 | 1分钟 | 立即 | 专注，最长1小时",    // 可选，如何描述都可以
"专注": true/false,                                     // 可选，默认 false
"豁免": {
  "属性": "体质",
  "DC": 14,
  "成功效果": "伤害减半",
  "失败效果": "正常生效"
},                                                       // 可选，无豁免则为 null
"描述": "你的皮肤变得粗糙，AC+2。"                      // 可选，功能外的补充文本
  }
]
```


### 效果类型


| 类型   | 说明          | 示例数值             |
| ---- | ----------- | ---------------- |
| `恢复` | 恢复生命值或法术位   | `2d4+2`, `1环法术位` |
| `伤害` | 对目标造成伤害     | `1d6`, `4d6毒素`   |
| `增益` | 暂时提升某属性或检定  | `+2`, `优势`       |
| `减益` | 暂时削弱        | `劣势`, `-1d4`     |
| `施法` | 施展一个法术（如卷轴） | `法术：火球术`         |
| `解除` | 解毒、治疗疾病等    | `解除中毒`           |


**没有效果（如普通面包在 D&D 中通常无效果）**：

- 建议 **不归入消耗品**，而是归入 `工具` 或 `杂物`。
- 如果你仍想保留在消耗品中，`效果` 字段写 `[]`（空数组），并在简介中说明“只能填饱肚子，无任何游戏效果”。

## 四、关于“常见物品列表”的规模

如果你只包括 **基础且常见的消耗品**（即 D&D 中几乎所有冒险者都会买得起、用得到的），大约 **30～40 种**。  
如果加上非普通、稀有的药水和卷轴，会扩展到 **70～100 种**。

我个人建议你：  

1. 先列出 **30 种左右**的种子列表（包括治疗药水、解毒剂、口粮、炽火胶等）。
2. 给 AI 一个 **“特殊物品生成规则”**：
  - 如果场景需要一瓶未列出的药水（如“火焰抗性药水”），AI 需要参考同类物品的 `效果` 结构生成，但必须通过玩家的批准才能正式入库。

这样既能保持一致性，又留有灵活性。

## 五、总结与建议

- **消耗品的数据结构**：按照我提供的字段（名称、数量、简介、价格、效果、使用条件、稀有度、重量）来设计，可以覆盖 90% 的场景。  
- **效果字段**：必须使用 **结构化写法**（类型+目标+数值），不要纯文本。  
- **常见物品列表**：建议你准备一个 **30 种左右的种子列表**，让 AI 有参照，拒绝其自行编造物品。  
- **价格波动**：给 AI 一个策略：“基础价浮动 20% 以内是正常的，超出需要提供理由（如饥荒、战乱）”。
